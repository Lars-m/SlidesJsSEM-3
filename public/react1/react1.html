React vs Angular 1.x

  • Angular was a framework, React is only a view-library, so:
    • Don't look for concepts like Controllers, Services Factories, Filters etc.
    • Don't look for services like $HTTP. How you connect to a backend is "your problem"
  • This one you either going to love or hate:
    • Angular 1.x had a very CLEAR separation between HTML and JavaScript, with the ng-xx directives as the way to connect the two
    • React has code and markup in one place (in your JavaScript)
  • React has (like Angular 2) no two way binding
  • React is a lot like "back to JavaScript". You really need to be a skilled JS developer to do modern React :-)
  • Like Angular, React does not ship with a Router component, so you must choose one (there is a obvious candidate)

React - The Virtual Dom

Manipulating the DOM is a fundamental part of all modern, interactive web. DOM manipulation unfortunately, is much slower than plain JS-operations

React's solution to this problem, is a Virtual Dom. For every DOM object, there is a corresponding "virtual DOM object." A virtual DOM object is a representation of a DOM object, like a lightweight copy.

  • The entire virtual DOM gets updated
  • The virtual DOM gets compared to what it looked like before you updated it. React figures out which objects have changed
  • The changed objects, and the changed objects only, get updated on the real DOM
  • Changes on the real DOM cause the screen to change

React - Components

Components are the heart and soul of React. They let you split the UI into independent, reusable pieces, and think about each piece in isolation.

Conceptually, components are like JavaScript functions. They accept arbitrary inputs (called "props") and return React elements describing what should appear on the screen.

Component written as a ES6 Class
Use only when features from base class is needed (often state) 

class Welcome extends React.Component {
  render() {
    return (
         <h1>Hello, {this.props.name}</h1>
        )
  }}

Functional Component as a JS-function

function Welcome(props) {
  return <h1>Hello, {this.props.name}</h1>
}
This is a valid React component because it accepts a single "props" object argument with data and returns a React element

Important things to remember:

Always start component names with a capital letter. <div />, as an example, represents a DOM tag, but <Welcome /> represents a component.

The parentheses around the return statement are needed because of JavaScript's automatic semicolon insertion. Without them, JavaScript would ignore all following lines.
If the JSX is a one-liner (on the same line as the return statement) parentheses are not needed.

State

There are two types of data that control a component: props and state. props are set by the parent and they are fixed throughout the lifetime of a component.

For data that is going to change, we have to use state In general, you should initialize state in the constructor, and then call setState when you want to change it.

State 2

Important things to know about state in react

The only place you can set state directly is in the constructor

Outside the constructor you must set state via the method: this.setState(..);

this.setState({ showText: !this.state.showText });

Observe, the method takes an object as argument

Calling this.setState(..); will Force a re-render (redraw the Component)

The only (non-recommended) alternative way with (plain) React to force a re-render is by calling: forceUpdate()


External libraries like Mobx can simplify state-handling

React Forms

HTML form elements work a bit differently from other DOM elements in React, because form elements such as <input>, <textarea>, and <select> typically maintain their own state and update it based on user input.

In React, mutable state is typically kept in the state property of components, and only updated with setState().

React suggests we combine the two by making the React state be the

"single source of truth"

Then the React component that renders a form also controls what happens in that form on subsequent user input.

An input form element whose value is controlled by React in this way is called a:

"controlled component"
