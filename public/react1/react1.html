Single Page Application - Pros & Cons

Pros

  • SPA's is usually faster, as most resources (HTML+CSS+Scripts) are loaded only once (Only data is transmitted back and forth.
  • This is was HTML5 was Designed for
  • Off line APPLICATIONS, using local storage
  • Perfect for as a match Service Workers and Progressive Web Apps
  • Cool transitions between pages, when all code already is here
  • SPAs are easy to debug with Chrome
  • Backend's can easilly be reused by other clients (mobile apps)

Cons

  • Harder to make SEO optimization for a Single-Page Application
  • Slow initial download since client frameworks must be loaded by the client
  • It requires JavaScript to be present and enabled
  • SPA is less secure. Due to Cross-Site Scripting (XSS)
  • Memory leaks in JavaScript can cause powerful systems to slow down

React vs React Native

When you Google "React", be aware of; that a great deal of the hits you get is for React Native and not plain React

React Native is a way to build native mobile apps (IOS or Android) with React and not Hybrid Applications as with Ionic and Cordova

  • The Ionic and Cordova strategy toward mobile development is Write once, Use anywhere
  • React's strategy is very different Learn once, write anywhere, which is a consequence of being "true native"

We will only scratch React Native this semester, but you will build an app, and while doing this, once again see the true beauty of REST :-)

React - The Virtual Dom

Manipulating the DOM is a fundamental part of all modern, interactive web. DOM manipulation, unfortunately, is much slower than plain JS-operations

React's solution to this problem, is a Virtual Dom. For every DOM object, there is a corresponding "virtual DOM object." A virtual DOM object is a representation of a DOM object, like a lightweight copy.

  • The entire virtual DOM gets updated
  • The virtual DOM gets compared to what it looked like before you updated it. React figures out which objects have changed
  • The changed objects, and the changed objects only, get updated on the real DOM
  • Changes on the real DOM cause the screen to change
