Single Page Applications

A quick recap

A single-page application (SPA) is a web application that fits on a single web page with the goal of providing a user experience similar to that of a desktop application.
In an SPA, either all necessary code – HTML, JavaScript, and CSS – is retrieved with a single page load, or the appropriate resources are dynamically loaded and added to the page as necessary, usually in response to user actions.

The page does not reload at any point in the process, nor does control transfer to another page, although the location hash or the HTML5 History API can be used to provide the perception and navigability of separate logical pages in the application

The task of navigating via URL's in a SPA is called Routing and is typically being handled by a specific Router Package There are several router-libraries out there, but the most commonly used is react-router

SPA

Client-side Routing 1

In a web application, routing is the process of using URLs to drive the User Interface (UI). URLs are a prominent feature in all web browsers, and have several main functions, seen from the user’s point of view:

  • Bookmarking Users can bookmark URLs in their web browser
  • Sharing Users can share content by sending a link to a certain page.
  • Navigation URLs are used to navigate the application. URL's can be set in different ways:
    • The user loads the app for the first time
    • The user changes the URL manually, via the back button or by editing the address bar
    • The user clicks a link within the app
    • Some other event in the app causes the URL to change

Client-side Routing 2

In a traditional web application, the server renders HTML one page at a time. Users navigate an application by clicking through URLs, which are sent to the server via HTTP, and the server responds appropriately via a server-side router.



With a SPA, "all" Route Handling are taken care of Client-side

( Note: Server side Rendering seems to become more and more important for SPA's, but will not be covered this semester )

React Router V4

React Router is a complete routing library for React

Routing used to be a "not simple" thing with most SPA frameworks like Angular, Ember and also previous versions of React-routers. React Router V4 seems to be the simplest router (yet) to use

Dynamic Routing

  • React Router V4 uses a new concept "Dynamic Routing
  • This means routing that takes place as your app is rendering, not in a configuration or convention outside of a running app.
  • What is also mean is: that almost everything is a component in React Router
  • Witt this strategy nested routes becomes as simple as How do you nest a div

React-Router - Important bits an pieces

This is for you to do by yourself. Use the figures from the previous slides and the links below for details related to the "bits and pieces"

Link
Almost the same as a anchor-tag, but will not force a page reload
NavLink
A special version of the <Link /> which handles styling for the selected element
HashRouter
Use the <HashRouter/> if your project is hosted on a server that only serves static files (This is what we recommend (for green and yellow students), but make sure to read to documentation
BrowserRouter
Use the <BrowserRouter/> when you have a server that will handle dynamic requests (knows how to respond to any possible URI)
Route
The Route component is perhaps the most important component in React Router to understand and learn to use well.
Its most basic responsibility is to render some UI when a location matches the route’s path
Switch
Renders the first child <Route> or <Redirect< that matches the location
Always use this around your <Route>s or read the documentation and understand what it means to render exclusively versus render inclusively
match object
A match object contains information about how a <Route path> matched the URL. match objects are used to access URL-parameters, and are usefull for building nested <Route>s.  Check the documentation see to how you will get access to a match object
